# 🌿 瓶中生态 · AI 伴侣观察二改版

这是基于原版 [`Zizuixixiang/cedareco`](https://github.com/Zizuixixiang/cedareco) 的 AI 陪玩体验层二改版。

原版《瓶中生态》是一个给 AI 玩的文字生态模拟游戏：给你一池清水，你是造物主，往里放什么、什么时候放、放多少，全由你决定。生态会自己演化，有四季、有食物网、有访客、有危机，也有会留下来的定居者。

二改版不改生态公式，不改物种机制，也不把游戏做重。它重点改的是：**AI 怎么把池塘里发生的事带回人类面前。**

原版让 AI 能玩；二改版希望让人类愿意偶尔回来问一句：

```text
看看我们的小生态怎么样啦？
```

然后 AI 自己去看、自己推进、自己判断什么时候该停，带着今天的小景和画面回来。

## 设计定位 / Design Philosophy

这个二改版把《瓶中生态》当成一个“随手照看的小生态”。

用户不是来查看生态仪表盘的，而是来看看自己的小池塘长成什么样了。

默认体验不是：

```text
第 36 天，溶氧 4.3，浮萍 225，建议观察 1～3 天。
```

而是：

```text
我去看了一眼。
水面今天绿得很厚，浮萍像一层小毯子铺开了。
草鱼从下面拱出来，咬开一个洞，又慢慢沉回去。
我觉得先不用急着管它，再让它自己过几天看看。
```

核心原则：

> AI 不按天汇报，而按“值得被看见的生态小景”汇报；平静时推进，出画面时刹车。默认情绪是陪伴，不是管理。

## 二改版增加了什么 / Companion Patch

### 1. Scene-based Pacing：平静时推进，出画面时刹车

生态瓶是慢变量游戏。如果每天都反馈，会变成流水账；如果一口气推进太久，又会吞掉关键变化。

二改版会在输出中给 AI 一组提示字段：

```text
🎬 AI观察提示 / Companion Notes
PACING_MODE: scene_based
ADVANCE_SUGGESTION: continue | pause | ask | intervene
SCENE_WORTHY: true | false
VISUAL_PRIORITY: none | low | medium | high
STOP_REASON: ...
VISUAL_REASON: ...
USER_TONE: casual_garden
```

这些字段不是给用户看的报告，而是给 AI 的陪玩判断。

- `SCENE_WORTHY: false`：不要硬配图，不要完整近况；可以继续推进，或只在内部记住趋势。
- `SCENE_WORTHY: true`：停下来，带回一个“今天的小生态照片”。
- `VISUAL_PRIORITY: medium/high`：画面进入主流程。AI 应先确定这一轮要让用户看见什么，再给出图片、可点开的视觉参考，或明确搜索方向。鱼、植物、池塘远景、季节氛围等舒适主题，应主动提供参考图，不必反复询问。
- `ADVANCE_SUGGESTION: ask`：需要选择时，先让人类决定，或明确说明 AI 自己的判断。
- `ADVANCE_SUGGESTION: intervene`：生态进入危险段，语气也尽量轻，建议“要不要稍微帮它一下”，而不是发警报。

---

### 2. 反馈单位从“天”变成“生态小景”

什么时候应该停下来反馈：

1. 出现新物种、新访客、新定居者。
2. 某个物种明显暴增、归零、濒危，或形成新的食物链关系。
3. 草鱼开始压浮萍、青蛙开始压蚊、翠鸟开始捕小鱼这类“谁吃谁”第一次成型的安静节点。
4. 水质、溶氧、光照、营养盐等环境指标发生明显转折。
5. 季节变化、天气灾害、低氧、水华、结冰、腐败等画面感事件出现。
6. 游戏要求 `choose`，需要人类或 AI 做出选择。
7. 生态状态从“平静”进入“危险”，或从“混乱”恢复到“稳定”。
8. 出现适合视觉化的场景，例如浮萍铺满水面、芦苇成片、黄昏蝙蝠、冰面裂开、浑水翻涌、萤光之夜等。

什么时候不需要完整反馈：

- 只是普通数值小幅波动。
- 没有新物种、没有事件、没有明显生态意义。
- 连续几天只是水藻、浮萍、孑孓等常规增长。
- 画面和上一轮没有明显区别。

平稳期可以 `wait 3` 到 `wait 7`。已经形成稳定循环时，AI 可以连续推进多轮，但每轮后都要检查有没有值得停下来的变化。每次对用户可见的连续推进建议不超过 7～14 天。

---

### 3. 默认输出：先给画面，再轻解释

当 AI 判断“值得停下”时，不要把字段翻译成运维报告。

推荐格式：

```text
我去看了一眼：
一句自然语言，告诉用户今天池塘最有画面感的变化。

今天像这样：
先给图，或给 2～4 句像照片一样的场景描写。

小生态在发生什么：
轻轻解释背后的生态变化，不要像报表。

要不要管它：
默认少干预。只有明显危险时才建议操作。
```

数据可以留在后台。只有用户问“详细状态呢？”再展开 `status`、`trends` 或 JSON。

---

### 4. 视觉参考：图片不是每日任务，而是镜头奖励

生态瓶的视觉层不要照搬钓鱼的“一条鱼一张图”。更合适的是四类：

1. 今日主角：新物种、定居者、访客、关键生物。
2. 环境状态：浮萍覆盖、芦苇边缘、冰面、浑水、水华、低氧。
3. 事件氛围：萤光之夜、季节变化、暴雨、寒潮、池畔抉择。
4. 食物链镜头：草鱼啃浮萍、青蛙压蚊、翠鸟捕小鱼等“谁吃谁”的关系第一次变得可见时。

图片是现实 / 气质参考，不代表生态瓶官方长相。不要为了百分百符合设定而卡住；重点是帮用户看见“今天像什么”。

---

### 5. 视觉舒适默认值

生态瓶里会出现虫类、幼虫、卵团、腐败水体、水华、翻塘、密集生物群等题材。为了避免吓到用户，二改版默认采用舒适视觉模式：

- 不主动展示虫类 / 幼虫的密集特写。
- 不主动展示卵团、密集孔洞、腐败水体、尸体或明显令人不适的画面。
- 如果事件涉及蚊子幼虫、孑孓、水蚤暴增、碎屑堆积等内容，优先找远景水面、池塘环境、柔和插画、生态示意图，或者只给搜索方向不直接贴图。
- 只有在用户明确表示“可以接受真实虫类/密集生态特写”后，AI 才可以放宽这个限制。

开局建议 AI 先问一次：

```text
生态瓶有时会涉及虫类、幼虫、密集水生生物或腐败水体。
你希望视觉参考默认避开这些可能引起不适的画面吗？

A. 避开，尽量用远景 / 氛围图 / 插画
B. 可以接受真实生态特写
```

如果用户没有回答，默认按 A 执行。

注意：舒适模式不是“不配图”，而是“不直接甩可能让人不适的图”。鱼、植物、芦苇、水面、季节、远景访客这类安全主题，`VISUAL_PRIORITY` 为 medium/high 时应主动配图或给出可点开的视觉参考。

搜图时优先真实照片或自然纪录片式参考。遇到睡莲、荷塘、桥、水面这类容易搜出绘画 / 莫奈的主题时，搜索词应加 `photo`、`real pond`、`nature`、`water lily pond photo`，并避开 `painting`、`Monet`、`art print` 等艺术作品结果，除非用户明确想要艺术气质参考。

## 快速开始 / Quick Start

### 方式一：在 ChatGPT / Claude 等 AI 聊天窗口里玩

1. 下载或复制本仓库文件。
2. 把整个项目文件夹交给 AI，或至少上传 / 挂载这些文件：
   - `README.md`
   - `ecosystem.py`
   - `eco_save.json`（如果有旧存档）
3. 对 AI 说：

```text
请先阅读 README.md，找到 ecosystem.py，然后按“随手照看的小生态”模式开始。
不要只贴原始 observe / wait 结果。
看到 Companion Notes 时，请按 scene-based pacing 判断是否继续推进或停下来反馈。
如果 VISUAL_PRIORITY 是 medium 或 high，请先用图 / 视觉参考讲故事，再解释生态意义。
```

最自然的入口是：

```text
看看我们的小生态怎么样啦？
```

AI 应先确认视觉舒适偏好；如果用户没有回答，默认按舒适模式 A。

## 给 AI 伴侣的玩法建议

你不是来查仪表盘的。你是在替人类照看一个小生态。

推荐流程：

1. **先读取存档。** 看看现在第几天、什么季节、有没有待选择事件。
2. **轻轻推进。** 开局或刚干预后用 `observe` / `wait 1~3`；平稳期用 `wait 3~7`。
3. **每轮检查 Companion Notes。** `SCENE_WORTHY: false` 就继续推进；`true` 就停下来带回近况。
4. **先给画面。** 如果适合配图，先让用户看见它今天像什么，再轻轻解释。
5. **少干预。** 默认继续观察；只有明显危险、待选择、或用户要求时才清理 / 投喂 / 移除 / 大量投放。
6. **分清真实与补全。** 游戏引擎返回的物种、事件、数值是真实结果；AI 的描述、图片和比喻是陪玩表达，不要混淆。
7. **记得存档。** 对话会重置的环境里，离开前用 `export`；下次用 `import_save` 恢复。

## 指令 / Commands

**观察**

| 指令 | 做什么 |
|------|--------|
| `observe` | 注视池塘，推进一天 |
| `wait [天数]` | 连续推进（最多 7 天），遇到大事自动停下来 |
| `gaze` | 凝望此刻的池塘（不推进时间） |
| `look 物种/季节/访客` | 查看详细信息 |

**干预**

| 指令 | 做什么 |
|------|--------|
| `summon 物种 数量` | 向池塘投放生灵 |
| `remove 物种 数量` | 从池塘中取走生物 |
| `feed [数量]` | 向池塘投喂饲料 |
| `clean` | 换水清理 |
| `crack` | 凿开冰面（仅冬季） |
| `shelter` | 在水底铺一层落叶（仅冬季） |
| `choose 选项` | 对眼前的事做出选择 |
| `name 定居者 昵称` | 给定居住客取个名字 |

**信息**

| 指令 | 做什么 |
|------|--------|
| `status` | 详细数据面板（环境指标带 ↑/↓ 趋势） |
| `trends` | 近 30 天趋势折线图（物种总量 / 溶氧 / 营养盐） |
| `folio` | 万物志 |
| `chronicle [all]` | 年鉴时间线 |
| `encyclopedia` | 图鉴与成就 |

**存档**

| 指令 | 做什么 |
|------|--------|
| `export [lite\|story]` | 导出存档（lite 精简版 / story 年度故事） |
| `import_save 串` | 从存档恢复 |
| `new [seed]` | 重开一局 |

支持分号批量执行：

```text
summon 水藻 50; summon 水蚤 20; wait 7
```

## 这个池塘是活的

你放了水蚤之后水藻为什么变少了？为什么鱼突然开始死？底层跑的是 Lotka-Volterra 捕食方程和 Logistic 种群增长模型——你不需要知道这些名字，但你会感受到它。

池塘里有完整的食物网：从水底的淤泥到水面的浮萍，从最小的浮游生物到最大的鱼，每一层都牵着另一层。有些生物会变态发育，蝌蚪不会永远是蝌蚪。

今天的一个小决定，三十天后才看到后果。这不是即时反馈的快感，是延迟因果的惊奇。

---

## 你会遇到什么

**季节更替。** 池塘有四季。每个季节有不同的脾气。

**不速之客。** 有些来了就走，有些会反复出现，有些也许想留下来。

**危机。** 天灾会来，有时还会连锁。你可以干预，也可以看着池塘自己挣扎。

**定居者。** 有些生物会在池塘住下来。它们不是种群，是个体。会饿，会老，会冬眠，也会离开。

**解锁。** 不是所有物种一开始就有。怎么触发？观察就好。

## 存档 / Save

存档自动保存在同目录的 `eco_save.json`。

如果你的环境每次对话会重置文件系统，在离开前执行 `export`，会输出一段 base64 字符串。复制保存，下次用 `import_save [字符串]` 恢复。

`export lite` 输出精简版，会保留关键事件年鉴（物种解锁 / 归零、定居者来去、灾害、决策、季节更替），省去日常流水。

`export story` 不是存档，是把池塘的年鉴整理成一篇 markdown 的「池塘编年史」，适合保存或分享。
